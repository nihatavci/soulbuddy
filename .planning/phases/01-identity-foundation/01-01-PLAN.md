    Table `public.profiles`:
    - `user_id uuid primary key references auth.users(id) on delete cascade` (one row per user; also the unique owner key)
    - `alias text not null` with a case-insensitive uniqueness index (`create unique index profiles_alias_lower_key on public.profiles (lower(alias))`) and `check (char_length(btrim(alias)) between 3 and 20)`
    - `intent text[] not null default '{}'` with `check (array_length(intent,1) >= 1 and intent <@ array['date','slow_burn','conversation','maybe']::text[])`
    - `boundaries text[] not null default '{}'` with `check (array_length(boundaries,1) >= 1 and boundaries <@ array['no_explicit_content','not_looking_to_meet_yet','go_slow','text_only']::text[])`
    - `age_confirmed_at timestamptz` (nullable; non-null = declared adult; do NOT add a DOB column)
    - `onboarded boolean not null default false`
    - `trust_level smallint not null default 0` (scaffold only, unenforced this phase)
    - `created_at timestamptz not null default now()`, `updated_at timestamptz not null default now()`
    - an `updated_at` BEFORE UPDATE trigger (mirror the timestamp-trigger idea from the analog).

    RLS (the IDEN-04 spine): `alter table public.profiles enable row level security;` then three owner-only policies keyed on `auth.uid() = user_id`:
    - `create policy "profiles_select_own" ... for select using (auth.uid() = user_id)`
    - `create policy "profiles_insert_own" ... for insert with check (auth.uid() = user_id)`
    - `create policy "profiles_update_own" ... for update using (auth.uid() = user_id) with check (auth.uid() = user_id)`
    (no delete policy in v1). Do NOT use `(auth.jwt() ->> 'sub')` — that is the archived pattern.

    Column-limited view `public.public_profiles`:
    - `create view public.public_profiles with (security_invoker = true) as select user_id, alias, intent, boundaries from public.profiles;`
    - `revoke all on public.public_profiles from anon;` and `grant select on public.public_profiles to authenticated;`
    - The view MUST NOT join `auth.users` and MUST NOT select email, age_confirmed_at, trust_level, created_at, or updated_at.

    After applying, regenerate the client types via MCP `generate_typescript_types` and overwrite `services/database.types.ts` with the result (do NOT hand-edit it).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| RN client → Supabase PostgREST/Auth | Client holds only the anon (publishable) key; all input crossing here is untrusted |
| another user's client → public_profiles | The single readable window onto other users — must never leak identity |
| auth.users (email) ↔ public.profiles (alias) | The anonymity separation line: email lives only in auth.users, never selected by app queries |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | public.public_profiles view | mitigate | View selects only {user_id, alias, intent, boundaries}; `security_invoker`; granted to `authenticated`, revoked from `anon`; no `auth.users` join. Asserted by verify-anonymity.sql. HIGH — blocks phase. |
| T-01-02 | Elevation of Privilege | profiles row access | mitigate | Owner-only RLS `auth.uid() = user_id` on select/insert/update; no delete policy |
| T-01-03 | Spoofing | client credentials | mitigate | Only anon/publishable key in `EXPO_PUBLIC_*`; service-role key never in client (elevated writes go through Worker in later phases) |
| T-01-04 | Information Disclosure | anon role read | mitigate | RLS + `authenticated`-only grant on view; `anon` has no SELECT on profiles or public_profiles |
| T-01-05 | Tampering | intent/boundaries payload | mitigate | DB CHECK constrains array elements to the locked vocab and requires ≥1 |
| T-01-SC | Tampering | npm/pip/cargo installs | accept | No new packages added in Phase 1 — all dependencies pre-vetted in the boilerplate; brand fonts added as local .ttf assets, not npm |
</threat_model>
