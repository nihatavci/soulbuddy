## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| onboarding form → profiles (write) | Untrusted alias/intent/boundary input crosses into the DB |
| current user → public_profiles (read) | The window onto other users; must stay identity-free |
| MMKV age flag → profiles.age_confirmed_at | Age truth moves from device to server on first write |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-11 | Information Disclosure | public_profiles after real write | mitigate | Re-verified live: view exposes only {user_id, alias, intent, boundaries}; own-row-only via RLS `auth.uid() = user_id`. HIGH — blocks phase. |
| T-01-12 | Tampering | intent/boundary values | mitigate | Client restricts to locked vocab AND Plan 01 DB CHECK rejects out-of-vocab arrays; `onboarded` write goes through owner-only RLS |
| T-01-13 | Information Disclosure | alias content | mitigate | `validateAlias` reserved/profanity screen; no real-name field exists; case-insensitive uniqueness index |
| T-01-14 | Spoofing | writing another user's profile | mitigate | Upsert keyed on `auth.uid()`; RLS insert/update `with check (auth.uid() = user_id)` |
| T-01-15 | Information Disclosure | alias enumeration via uniqueness error | accept | Low value in closed beta; inline generic "alias is taken" copy |
| T-01-SC | Tampering | npm/pip/cargo installs | accept | No new packages added in this plan |
