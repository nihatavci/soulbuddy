## Design Context

### Users
Young adults (18-30) navigating dating apps. They open CoupleAI when they've matched with someone and need help crafting the right message — an opening line, a reply, or an ask-out. The context is private, slightly nervous, wanting to sound their best. The app should feel like a warm, witty coach in their corner — not a robotic text generator.

### Brand Personality
**Bold, witty, premium.**
Confident tone that empowers without being arrogant. Clever copy that feels human. High-end execution that earns trust. The app should make users feel like they have a secret advantage — not like they're cheating.

### Emotional Goals
- **Primary:** Warm and supportive — encouraging, non-judgmental, a coach in your corner
- **Secondary:** Confidence — user feels empowered, ready to make a move
- The interface should never feel desperate, clinical, or gimmicky

### Aesthetic Direction

**Visual tone:** Revolut / Linear — clean utility, polished micro-interactions, generous whitespace, zero visual clutter. Every pixel earns its place.

**Theme:** Light only. White canvas (#FFFFFF) with carefully placed accent color. No dark mode.

**Color philosophy:** Restrained. The accent coral (#FF4D6D) is used sparingly for CTAs, badges, and active states. Pastel card themes (coral, blue, pink, purple, green, gold) add warmth without overwhelming. Too much color destroys consistency — use it like seasoning, not paint.

**Typography:** Plus Jakarta Sans throughout. Bold weight for headings, Medium for body. Clean, geometric, modern. No decorative fonts in core flows.

**Spacing:** Golden ratio system (phi = 1.618, base 4px). Every margin and padding uses tokens from the Space scale. No raw numbers. Consistency in spacing is non-negotiable.

**Icons:** Feather icon set (primary) + Ionicons (secondary). Never emojis. When an icon is needed, it must be high quality — bare text is preferable to a bad icon, but a good icon is always better than text alone.

**Buttons:** Cinematic pill-shaped buttons — generous padding, high contrast, icon + text side by side, accent shadows. Reference: Sobers app style. The Button component supports primary/secondary/ghost/dark variants.

**Animations:** Spring-press on touchables (scale 0.97, spring back). ChromaRing (Skia shader) for processing states. FadeInDown for content entry. Keep animations purposeful — they signal state change, not decoration.

### Anti-References
- **Generic AI chatbot** — No ChatGPT vibes. Not a blank chat screen with a cursor.
- **Cheap dating app** — No neon gradients, no swipe-card gimmicks, no tacky hearts everywhere.
- **Over-designed / busy** — No excessive animations, parallax, or visual noise competing for attention. If it doesn't serve the user's task, remove it.
- **Inconsistent** — The #1 anti-pattern. Every screen must feel like it belongs to the same app. Same spacing tokens, same typography, same color usage patterns.

### Design Principles

1. **Consistency is king.** Same spacing tokens, same font weights, same color patterns on every screen. The challenges page is the benchmark — clean, well-spaced, pastel cards with clear hierarchy. Every new screen should feel like a sibling, not a cousin.

2. **Whitespace is a feature.** Generous padding and margins. Let content breathe. Revolut-level cleanliness. When in doubt, add more space, not more elements.

3. **Color with restraint.** White background, one accent color for interactive elements, pastels for cards. Never use color for decoration — only for meaning (active state, category, status). Too much color is worse than too little.

4. **Icons over emojis, always.** Feather/Ionicons over emoji, every time. But only use an icon when it adds clarity. An icon that doesn't help is visual noise.

5. **Motion with purpose.** Spring-press feedback on every touchable (users expect it). ChromaRing for processing. FadeInDown for content entry. No gratuitous animation. If removing an animation doesn't hurt comprehension, remove it.
